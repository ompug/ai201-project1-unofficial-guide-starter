Title: Student Notes - Ingestion and Cleaning
Source: Local file description: peer implementation note based on Document Ingestion Pipeline requirements
Date: 2026-06-10
Category: ingestion
Rating: 4

The ingestion script should load every raw document into memory, clean it, and preserve useful metadata. The cleaning step should remove HTML tags, navigation text, extra whitespace, repeated headers, cookie banners, ads, share buttons, and unrelated boilerplate. It should keep the actual review text, ratings, opinions, and source context.

For this local corpus, each file starts with metadata lines such as Title, Source, Date, Category, and Rating. Those lines are parsed into metadata fields and the remaining body text becomes the document content. Keeping category and rating as metadata allows metadata filtering later.

After cleaning, print one complete document and read it. If the output still contains HTML entities, navigation fragments, or broken text, fix cleaning before chunking. Bad cleaning produces bad chunks, and bad chunks make retrieval and generation fail even when the model is strong.
