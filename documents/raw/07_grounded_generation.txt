Title: Student Notes - Grounded Generation and Source Attribution
Source: Local file description: peer implementation note based on grounded generation rubric
Date: 2026-06-10
Category: generation
Rating: 5

Grounded generation means the answer must use only retrieved chunks. The system prompt should explicitly say that if the answer is not supported by the retrieved context, the assistant must refuse and say the corpus does not contain enough information. This prevents the model from filling gaps with general knowledge.

Every answer should show source attribution in the output text. A simple pattern is to cite bracketed source names after claims, such as [04_chunking_strategy.txt] or [06_retrieval_semantic_hybrid.txt]. Citations should point to the documents used by the answer, not just any document that happened to be retrieved.

The pipeline should also use a relevance threshold. If the top retrieved chunks have very low scores or the query is obviously outside the domain, the response should refuse instead of guessing. An out-of-scope example is "What is the housing deposit deadline for UCLA?" because that fact is not in the AI201 Project 1 corpus.
