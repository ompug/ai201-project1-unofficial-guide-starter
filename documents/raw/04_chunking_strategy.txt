Title: Student Notes - Chunking Strategy for Short Advice Documents
Source: Local file description: peer implementation note based on Milestone 3 chunk inspection guidance
Date: 2026-06-10
Category: chunking
Rating: 5

For short advice files, chunks should be large enough to preserve a complete recommendation but small enough that unrelated topics do not get blended. A target of about 900 characters works well for these documents because most important points fit in one to three paragraphs. That size gives semantic search enough vocabulary to match questions without burying the answer inside a huge chunk.

An overlap of about 180 characters protects transitions between paragraphs. If a key sentence mentions a requirement at the end of one chunk and the reasoning at the start of the next, the overlap lets either chunk remain understandable. Too little overlap can split context; too much overlap creates duplicate search results.

The chunker should prefer paragraph boundaries before using a character limit. It should remove repeated boilerplate, normalize whitespace, and keep source metadata with every chunk. Five labeled sample chunks should appear in the README with the source document name.
