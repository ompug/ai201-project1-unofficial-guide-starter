Title: Student Notes - Common Rubric Point Traps
Source: Local file description: peer checklist based on the grading rubric supplied with the assignment
Date: 2026-06-10
Category: rubric
Rating: 5

Several rubric points are easy to miss even when the code works. The README needs at least five labeled sample chunks with the source document name. It also needs at least three retrieval test examples, and at least two of those examples need written explanations of why the returned chunks are relevant.

The grounded generation section needs at least two example system responses with visible source attribution. It also needs one out-of-scope query example with the refusal response shown. The README must explain how grounding is enforced in the prompt or pipeline.

The spec reflection is required for README completeness. It should name one way the spec helped and one divergence from the spec with a reason. The planning.md file also has separate completeness points, so it should not be treated as optional once the code works.
