Title: Student Notes - planning.md Must Be Written First
Source: Local file description: peer planning note based on Milestone 2 instructions
Date: 2026-06-10
Category: planning
Rating: 5

planning.md should be completed before pipeline code. It needs substantive sections for Domain, Documents, Chunking Strategy, Retrieval Approach, Evaluation Plan, Anticipated Challenges, Architecture, and AI Tool Plan. Placeholders such as "TBD" or one-line answers are risky because the grader awards points for specificity.

The Evaluation Plan must list five specific test questions with verifiable expected answers. A question like "What should I know about retrieval?" is too open-ended. A stronger question is "How many retrieval test examples must the README include?" because the expected answer is exactly three.

The Architecture section should include a simple diagram with the five stages: Document Ingestion, Chunking, Embedding plus Vector Store, Retrieval, and Generation. Each stage should name the tool or library used, such as sentence-transformers for embeddings and ChromaDB for the vector store.
