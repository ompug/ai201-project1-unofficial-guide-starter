The project should name the embedding model. The recommended free local model is sentence-transformers/all-MiniLM-L6-v2. It is small, fast, and runs locally, which avoids API cost and rate limits. A production system might consider a larger model for higher accuracy, longer context, multilingual support, or better handling of domain-specific wording, but that can increase latency and cost.

Retrieval should return the top relevant chunks for a query. Top-k equals five in this project because five chunks usually provide enough context for a grounded answer without overwhelming the generator. Too few chunks can miss supporting evidence; too many chunks can add distracting context.

Hybrid search combines semantic similarity with BM25 keyword matching. Semantic search helps when the query uses different words from the document. BM25 helps when the query contains exact rubric terms such as "top-k", "planning.md", or "sample chunks". A weighted score can combine 65 percent semantic score and 35 percent BM25 score.
